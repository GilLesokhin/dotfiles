# MISSION.md Format

`MISSION.md` lives at the workspace root. It captures why the user is learning the topic. Every teaching decision should trace back to it.

```md
# Mission: {Topic}

## Why
{1-3 sentences describing the concrete real-world goal and what changes when they have this skill. Avoid abstract framings.}

## Success looks like
- {A specific, observable thing the user will be able to do}
- {Another specific thing}

## Constraints
- {Time, budget, prior commitments, learning preferences, or other bounds}

## Out of scope
- {Adjacent topics the user does not want to pursue now}
```

- Maintain one mission per workspace; unrelated topics need separate workspaces.
- Prefer concrete goals over abstract ones.
- Interview the user before writing a vague mission.
- Update the mission when reality shifts.
- Keep it to one screen.
