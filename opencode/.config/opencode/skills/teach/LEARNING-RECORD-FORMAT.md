# Learning Record Format

Learning records are stored in `learning-records/` and named sequentially: `0001-slug.md`, `0002-slug.md`, and so on. Create the directory only when writing its first record.

They capture non-obvious learning, key insights, and prior knowledge that should steer future sessions and determine the zone of proximal development.

```md
# {Short title of what was learned or established}

{1-3 sentences explaining what was learned or established and why it matters for future sessions.}
```

Add optional status frontmatter (`active` or `superseded by LR-NNNN`), evidence, or implications only when useful.

Write a learning record when the user demonstrates non-trivial understanding, reports prior knowledge, corrects a misconception, or changes the mission. Do not write one merely because material was covered, for glossary definitions, or as a session log.

When a later record contradicts an earlier record, mark the old one as superseded rather than deleting it.
