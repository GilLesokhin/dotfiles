# GLOSSARY.md Format

`GLOSSARY.md` defines canonical language for the teaching workspace. All lessons and learning records should use its terminology.

```md
# {Topic} Glossary

{One or two sentences describing the topic.}

## Terms

**Term**:
A concise definition.
_Avoid_: aliases or misleading terms
```

- Add a term only after the user understands it.
- Pick preferred terminology and list aliases to avoid.
- Keep definitions to one or two sentences.
- Reuse glossary terms inside later definitions.
- Group terms under subheadings when useful.
- State any important ambiguity explicitly.
- Revise definitions as understanding deepens.
